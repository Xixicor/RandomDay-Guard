# Evidence Boundary

RandomDayGuard is Saved-folder and log-derived only.

It can report:

- server log Login request identity fields
- mapped joins/leaves
- clean and unclean session closure
- server lifecycle context
- warning/failure bursts visible in logs
- visible object/class/path strings in readable Saved files
- object registry and world-state summaries from extracted strings

It does not claim:

- exact live coordinates
- direct item duplication
- direct object damage
- direct container use
- item theft
- inventory actions
- object ownership
- live ping
- memory inspection
- client anti-cheat behavior
- player-caused crash/restart/failure

Those facts may only be reported if readable server logs or Saved evidence directly expose them. Otherwise they remain limitations.

`UpdateActorToWorldSave` is a world actor save-touch or spatial context marker. It is not a warning by itself and is not proof of player action.
